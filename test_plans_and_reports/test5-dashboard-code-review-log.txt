BUGS FOUND
===================================================================

BUG-1 [FIXED] — Pagination display shows "Showing 1 - 0 of 0" when empty
  File: apps/dashboard/src/main.tsx, line ~533
  Severity: Low (cosmetic)
  Description: When accountsTotal=0 and accountsList is empty,
    the range formula ((page-1)*pageSize)+1 = 1 and
    Math.min(page*pageSize, 0) = 0, producing "Showing 1 - 0 of 0".
  Fix: Guard with conditional — show "No results" when total=0.

BUG-2 [FIXED] — handleAccountsSearch uses stale page state
  File: apps/dashboard/src/main.tsx, line ~223-226
  Severity: Medium (wrong data shown briefly)
  Description: handleAccountsSearch() called setAccountsPage(1) then
    immediately called loadAccountsList(). Since setState is async,
    loadAccountsList (useCallback closure) still read the old page value.
    This sent a request with the old page before the useEffect re-fired
    with page=1. If the user was on page 3 and searched, they'd briefly
    see page-3 results before getting page-1 results.
  Fix: When already on page 1, call loadAccountsList directly.
    When on other pages, only setAccountsPage(1) — let the useEffect
    handle the reload after state settles.

BUG-3 [NOT FIXED — intentional design] — accountsPlateFilter missing from useEffect deps
  File: apps/dashboard/src/main.tsx, line 137-139
  Severity: Low (by design)
  Description: The effect that reloads accounts on filter changes does not
    include accountsPlateFilter. Plate search is button-triggered only.
    Changing plate text does not auto-search. This is acceptable UX.
  Decision: Left as-is. Added eslint-disable comment for clarity.

BUG-4 [NO FIX NEEDED] — VerifyQueueSection props loading/error not passed
  File: apps/dashboard/src/main.tsx, lines ~578-587
  Severity: Low
  Description: VerifyQueueSection accepts optional loading and error props
    but main.tsx never passes them. The component uses defaults (loading=false,
    error=undefined). No crash — queue errors surface in console.error only.
  Decision: Acceptable. The component gracefully falls back to defaults.

BUG-5 [NO FIX NEEDED] — ImportSummarySection props loading/error not passed
  Same pattern as BUG-4. Acceptable.

BUG-6 [NOT A BUG — confirmed] — fetchAccountsSummary vs fetchAccount URL collision
  GET /api/v1/accounts/summary could potentially match
  GET /api/v1/accounts/{plate_text} with plate_text="summary".
  FastAPI declares /accounts/summary (line 174) before /accounts/{plate_text}
  (line 206), so the literal route wins. Safe.
